# Устранение проблем с деплоем админ-панели

## Типичные проблемы и их решения

### 1. Ошибки сборки (build errors)

**Симптом**: В Telegram-уведомлениях приходит сообщение об ошибке сборки админ-панели.

**Решение**:
- Проверьте commit history, чтобы увидеть последние изменения
- Локально выполните `npm run build` в директории `admin/` для воспроизведения ошибки
- Исправьте найденные ошибки и сделайте новый коммит

### 2. Проблемы с nginx

**Симптом**: Админ-панель недоступна, хотя сборка прошла успешно.

**Решение**:
- Проверьте конфигурацию nginx: `sudo nginx -t`
- Проверьте, что символическая ссылка создана:
  ```bash
  ls -la /etc/nginx/sites-enabled/ | grep admin
  ```
- Проверьте логи nginx: `sudo tail -n 100 /var/log/nginx/error.log`

### 3. Проблемы с SSL-сертификатом

**Симптом**: В браузере сообщение о недействительном сертификате или его отсутствии.

**Решение**:
- Проверьте статус сертификата:
  ```bash
  sudo certbot certificates
  ```
- Обновите сертификат, если он истек:
  ```bash
  sudo certbot --nginx -d admin.unlim-mind.ai
  ```

### 4. Проблемы с доступом к директории деплоя

**Симптом**: В логах ошибка "Permission denied" при копировании файлов.

**Решение**:
- Проверьте права на директорию:
  ```bash
  ls -la /var/www/
  ```
- Измените владельца директории, если необходимо:
  ```bash
  sudo chown -R www-data:www-data /var/www/admin-panel
  sudo chmod -R 755 /var/www/admin-panel
  ```

### 5. Ошибки маршрутизации (404 при обновлении страницы)

**Симптом**: При прямом доступе к маршрутам (например, `/users`) возникает ошибка 404.

**Решение**:
- Убедитесь, что в конфигурации nginx есть правило для перенаправления всех запросов на index.html:
  ```
  location / {
      try_files $uri $uri/ /index.html;
  }
  ```

### 6. Проблемы с API-запросами

**Симптом**: Админ-панель загружается, но данные не отображаются.

**Решение**:
- Проверьте, что проксирование API настроено правильно в nginx
- Проверьте логи API на наличие CORS-ошибок
- Убедитесь, что API-сервер работает: `systemctl status unlim-api`

### 7. Проблемы с Telegram-уведомлениями

**Симптом**: Уведомления не приходят в Telegram.

**Решение**:
- Проверьте, что скрипт `telegram.py` имеет правильные права:
  ```bash
  chmod +x /var/www/unlim-mind-ai/tools/notifier-bot/telegram.py
  ```
- Проверьте доступность API Telegram с сервера:
  ```bash
  curl https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getMe
  ```

## Проверка текущего статуса

Выполните следующую команду для проверки статуса всех компонентов:

```bash
cd /var/www/unlim-mind-ai
python3 tools/system-status.py
```

## Ручной перезапуск сервисов

```bash
sudo systemctl restart nginx
sudo systemctl restart unlim-api
```
