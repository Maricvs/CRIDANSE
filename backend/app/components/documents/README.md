# Компонент для векторизации документов и семантического поиска

Этот компонент реализует функциональность для работы с документами, их векторизации и семантического поиска в проекте Unlim Mind AI.

## Основные возможности

1. **Загрузка документов**:
   - Поддерживаемые форматы: PDF, DOCX, TXT
   - Документы сохраняются в файловой системе
   - Метаданные сохраняются в базе данных
   - Документы привязываются к пользователю

2. **Векторизация документов**:
   - Автоматическое разбиение документов на части (chunks)
   - Создание эмбеддингов с помощью OpenAI API
   - Сохранение эмбеддингов в базе данных

3. **Семантический поиск**:
   - Поиск релевантных частей документов по запросу
   - Вычисление косинусного сходства между запросом и эмбеддингами
   - Возврат наиболее релевантных фрагментов

4. **Интеграция с режимом учителя**:
   - Автоматическое использование релевантных документов для ответов учителя
   - Контекстно-зависимые ответы на основе загруженных материалов

## API Endpoints

### Документы

- `POST /api/document_search/documents` - Загрузка нового документа
- `GET /api/document_search/documents` - Получение списка документов пользователя
- `DELETE /api/document_search/documents/{document_id}` - Удаление документа

### Поиск

- `POST /api/document_search/documents/search` - Семантический поиск по документам

### Режим учителя с контекстом

- `POST /api/teacher/ask` - Задать вопрос учителю с использованием контекста из документов

## Пример использования

1. Пользователь загружает учебные материалы
2. Система автоматически обрабатывает документы и создает эмбеддинги
3. Когда пользователь задает вопрос в режиме учителя, система:
   - Находит релевантные части документов по семантическому поиску
   - Включает эти части в контекст запроса к GPT
   - Генерирует ответ на основе загруженных материалов

## Настройка PostgreSQL для работы с pgvector

Для использования векторных операций в PostgreSQL необходимо установить расширение pgvector. Выполните следующие шаги:

1. Установите расширение:
```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

2. Создайте схемы в базе данных:
```sql
CREATE SCHEMA IF NOT EXISTS documents;
```

3. При первом запуске приложения таблицы будут созданы автоматически.

## Технические детали

- Документы разбиваются на части примерно по 1000 слов с перекрытием в 200 слов
- Эмбеддинги генерируются с помощью модели `text-embedding-3-small` от OpenAI
- Эмбеддинги сохраняются в формате JSON в базе данных
- Семантический поиск использует косинусное сходство для определения релевантности
